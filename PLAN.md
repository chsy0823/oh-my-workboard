# Plan — oh-my-workboard

Ported from an in-house team workboard (`prdv-tasks`) into a reusable Claude Code plugin.
Incorporates the tree+tags data model refactor and the `team.yaml` Single-Source-of-Truth decision.

## Core design

- **Task repo is separate from project repos.** The plugin stores the path to the task repo in `~/.claude/workboard.json`. Slash commands resolve the path and `cd` into the task repo before acting. This lets users run `/oh-my-workboard:start-day` from any project repo and have it operate on the shared workboard.
- **Team config SSOT is `.workboard/team.yaml`** inside the task repo. CLAUDE.md's team table, Slack workflow ID mapping, and per-member file scaffolds are all rendered from this file. Hooks parse `team.yaml` with `yq` (fallback `grep`/`awk`).
- **Hooks are path-scoped** — ownership and branch protections only fire inside the configured task repo path.
- **Solo mode = team mode with 1 member.** The only difference is whether a shared remote exists.
- **Workboard format is a single tree with tags**, not a bag of sections. Commands and templates assume this model.

## Data model

### Team SSOT — `.workboard/team.yaml`

```yaml
leader: chsy0823
members:
  - id: chsy0823
    name: 최수용
    keywords: [elenore, chsy, suyong, 최수용]
    role: Product Development Lead
    slack_id: U000000000
    active: true
  - id: example-member
    name: Example Member
    keywords: [example]
    role: Example Role
    slack_id: U000000001
    active: true
```

Consumers:
- `hooks/.claude/hooks/check-permissions.sh` → matches `git config user.name` against `members[].keywords + id + name`; leader bypass from `leader` field.
- `.githooks/pre-push` → same leader check.
- `CLAUDE.md` team table → rendered view (regenerated by `init` and `sync-team`).
- `people/{id}.md` scaffolds → created/pruned based on membership.
- `.github/workflows/slack-notify.yml` → `slack_id` mapping rendered in.

Adding a member = edit `team.yaml` → `/oh-my-workboard:sync-team` re-renders derived files.

### Workboard — `people/{id}.md`

One section per file:

```
## 이번주 작업 (W{ISO week})

- [ ] [project] top-level task (weekly-goal unit)
  - [x] subtask @done(YYYY-MM-DD)
  - [ ] subtask @today
  - [ ] subtask @wait(external reason, no user field)
  - 체크박스 없는 bullet = note (artifact link, conclusion, etc.)
- [x] [project] single-shot task @done(YYYY-MM-DD) @handoff(user: what's next)
```

Reserved tags:

| Tag | Meaning |
|-----|---------|
| `@today` | Picked for today |
| `@done(YYYY-MM-DD)` | Completed |
| `@wait(reason)` | Waiting on external / self-precondition — no `user:` field |
| `@block(reason)` | Same as `@wait` but higher severity |
| `@handoff(user: action)` | Someone else picks this up next |

### Cross-member dependencies — `board/requests.md`

Single queue with four categories: `PR 리뷰`, `디자인 리뷰`, `작업 요청`, `결정 필요`.
Ping-pong states: `🔄 수정요청 → @{requester} 반영중`, `🔄 재리뷰대기`.

**No duplication** between `board/requests.md` and individual workboards — a queued request does not get a `@today` leaf on the receiver's workboard. Briefings aggregate requests.md automatically.

## User config — `~/.claude/workboard.json`

```json
{
  "mode": "team",
  "path": "/Users/me/work/prdv-tasks",
  "remote": "chsy0823/prdv-tasks"
}
```

`mode` ∈ {`solo`, `team`}. `remote` optional; when present, commands `git push` after committing.

Team/leader/projects are **not** stored here — they live in the task repo's `.workboard/team.yaml`.

## Phases

### P0 — Repo bootstrap ✓
- `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json`
- Empty `commands/`, `hooks/`, `templates/`
- README + this PLAN

### P1 — Port the 4 existing commands
- `start-day`, `end-day`, `start-week`, `plan-week`
- Each command:
  - Reads `~/.claude/workboard.json`, `cd`s to the task repo
  - Resolves the current user via `git config user.name` against `.workboard/team.yaml`
  - Uses the tree+tags model (no references to the old 6-section format)
- Command-specific behavior:
  - `start-day`: detects previous `end-day` gap, briefs the W-tree (remaining top-level, `@today`, `@wait`), queries requests.md for incoming + ping-pong changes, reconciles `@wait` entries, then helps pick `@today` leaves (splitting oversized leaves into subtasks in place). Enforces the workboard↔requests.md no-duplication rule.
  - `end-day`: walks every `@today` leaf (done / partial / missed), adds `@done(D)` with completion notes per task type, splits partial leaves, auto-checks parents when all subs are done, asks for `@handoff`, captures off-plan work with "why" and adds a new leaf (project subtree or `[misc]` bucket).
  - `start-week`: snapshots the W{N-1} subtree into `log/w{N-1}.md` under `## {id}`, carries over unchecked top-level leaves, adds 2–4 new top-levels (no premature subtask splitting).
  - `plan-week`: leader-only (gated by `team.yaml:leader`); reviews every member's W{N-1}, updates milestones, snapshots the leader's own W{N-1}, writes `board/status.md` for W{N}.

### P2 — `/oh-my-workboard:init` (central)
- Interactive setup:
  - Mode: `--solo` / `--team-init` / `--team-join`
  - Team members (id, name, keywords, role, slack_id)
  - Team leader pick (→ `team.yaml:leader`)
  - Initial projects (optional)
  - Slack channel (optional — needs `slack_id` per member for workflow rendering)
  - Dashboard (optional)
  - Local path for the task repo
  - GitHub remote (optional, available in both modes)
- Actions:
  - Create folder tree (`board/`, `people/`, `projects/`, `log/`, `.claude/hooks/`, `.workboard/`)
  - Write `.workboard/team.yaml` from collected data
  - Render from `team.yaml`:
    - `CLAUDE.md` (team table, tree+tags model docs, requests.md rules, file layout, commit prefixes)
    - `people/{id}.md` per member with `## 이번주 작업 (W{current})`
    - `.github/workflows/slack-notify.yml` (slack_id mapping block, opt)
  - Copy unconditionally:
    - `board/{status,blockers,requests}.md` (requests.md with 4 empty categories)
    - `.githooks/{commit-msg,pre-push}` + `git config core.hooksPath .githooks`
    - `.claude/hooks/{check-permissions,remind-commit}.sh`
    - `.claude/settings.json`
  - (opt) Copy `dashboard/`
  - Write `~/.claude/workboard.json`
  - Initial commit + (opt) push
- **`--team-join` variant**: clones an existing task repo, skips rendering, just writes `~/.claude/workboard.json` after validating `.workboard/team.yaml` exists.

### P3 — Hooks (external scripts, team.yaml parsing)
- `hooks/hooks.json` declares plugin-level hooks (PreToolUse/PostToolUse/SessionStart) gated on `cwd == workboard.path`. Inside the task repo, they delegate to `.claude/hooks/*.sh`.
- `check-permissions.sh`:
  - Reads `.workboard/team.yaml`
  - Finds current user's id via keyword match against `git config user.name`
  - Enforces: non-leader can only edit their own `people/{id}.md`; non-leader cannot touch `projects/**` or `CLAUDE.md` on `main`
  - Uses `yq` if available, else `awk`/`grep` fallback for `leader` and `members[].id/keywords` extraction
- `remind-commit.sh` triggers on `people/**` edits.
- `.githooks/pre-push` reads `leader` from `team.yaml` to decide bypass.

### P4 — New commands
- `/oh-my-workboard:status` — read-only briefing from any cwd. Reuses the `start-day` briefing block without mutation.
- `/oh-my-workboard:where` — print configured task repo path, mode, remote.
- `/oh-my-workboard:sync-team` — re-render CLAUDE.md team table, Slack workflow ID mapping, and `people/{id}.md` scaffolds from `.workboard/team.yaml`. Prunes files for members flipped to `active: false` (with confirmation — archives instead of deletes).

### P5 — Template set
- `templates/team.yaml` — placeholder structure for `init` to fill
- `templates/CLAUDE.md` — Markdown template with render markers for the team table block; includes tree+tags docs, requests.md rules (incl. no-duplication), file layout, commit prefixes
- `templates/board/status.md`, `blockers.md`, `requests.md` (4 empty categories)
- `templates/people/_member.md` — `# {id} — {role}` + empty `## 이번주 작업 (W{N})`
- `templates/projects/_example/overview.md`, `milestones.md`
- `templates/.githooks/commit-msg` — prefix check for `[me]|[log]|[board]|[project]|[decision]|[init]`
- `templates/.githooks/pre-push` — leader bypass via `team.yaml`
- `templates/.claude/hooks/check-permissions.sh` — ownership + branch protection driven by `team.yaml`
- `templates/.claude/hooks/remind-commit.sh`
- `templates/.claude/settings.json` — wires the two hooks + SessionStart for dashboard
- `templates/.github/workflows/slack-notify.yml` — shell block renders `SLACK_ID_*` from `team.yaml`
- `templates/scripts/setup.sh` — one-time initializer (git hooks path, sanity checks, yq detection)
- `templates/dashboard/` — optional
- `templates/log/.gitkeep`

### P6 — Local tests
- `claude --plugin-dir ./oh-my-workboard`
- Three empty scratch dirs: solo / team-init / team-join end-to-end
- Run each command against a freshly initialized task repo
- Hooks fire only inside the task repo, not in sibling project repos
- Add a 6th member via `team.yaml` edit + `/oh-my-workboard:sync-team` — CLAUDE.md table, Slack workflow, and `people/` all update; ownership rule still works
- Flip a member to `active: false` — sync-team archives their file without losing history

## Decisions (locked)

- **Name:** `oh-my-workboard` (namespace `/oh-my-workboard:*`)
- **GitHub owner:** `chsy0823`
- **Marketplace name:** `workboard`
- **Slack + dashboard:** optional, asked during `init`
- **Team mapping SSOT:** `.workboard/team.yaml` inside the task repo. CLAUDE.md table, Slack workflow, and `people/` scaffolds are derived.
- **Parser:** `yq` primary, `awk`/`grep` fallback (no hard dependency in v0.1)
- **Workboard format:** tree + tags (dropped the 6-section format)
- **Cross-member dependencies:** `board/requests.md` only — no `@wait(user:...)` in workboards
- **Multi-team support:** single team in v0.1, array of contexts in a later version

## Deferred

- `P8` — migrate the existing `prdv-tasks` repo to consume this plugin (after v0.1 ships and is validated end-to-end)
- Multi-team context switching (`/oh-my-workboard:switch {name}`)
- Dashboard as its own separately-versioned plugin
